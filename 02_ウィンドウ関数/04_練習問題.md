## 練習問題
- フレーム句に何も指定していない場合はテーブル全体が一つの大きなグループとみなされる  
- PARTITION BYありでORDER BYなしだと、パーティションは区切られるが、並び替えの順序が不明のため、パーティション全体に集約関数が適用される
``` SQL
SELECT
	server_char
	,sample_date
	,SUM(load_val) OVER() as sumload
FROM
	Chapter2LoadSample;

-- result
server_char	sample_date	sumload
A	2018-02-01	74448
A	2018-02-02	74448
A	2018-02-05	74448
A	2018-02-07	74448
A	2018-02-08	74448
A	2018-02-12	74448
B	2018-02-01	74448
B	2018-02-02	74448
B	2018-02-03	74448
B	2018-02-04	74448
B	2018-02-05	74448
B	2018-02-06	74448
C	2018-02-01	74448
C	2018-02-07	74448
C	2018-02-16	74448


SELECT
	server_char
	,sample_date
	,SUM(load_val) OVER (
		PARTITION BY server_char) as sum_load
FROM
	Chapter2LoadSample

-- result
server_char	sample_date	sum_load
A	2018-02-01	8521
A	2018-02-02	8521
A	2018-02-05	8521
A	2018-02-07	8521
A	2018-02-08	8521
A	2018-02-12	8521
B	2018-02-01	62427
B	2018-02-02	62427
B	2018-02-03	62427
B	2018-02-04	62427
B	2018-02-05	62427
B	2018-02-06	62427
C	2018-02-01	3500
C	2018-02-07	3500
C	2018-02-16	3500
```
